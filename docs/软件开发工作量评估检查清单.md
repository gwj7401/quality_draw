## 📊 过程监督检查项

### 代码提交记录检查

- [ ] 获取Git仓库访问权限
- [ ] 检查提交频率是否合理
- [ ] 检查提交时间分布
- [ ] 检查代码变更量统计

**常用Git命令：**
```bash
# 查看时间段内的提交统计
git log --since="开始日期" --until="结束日期" --oneline --shortstat

# 查看每人提交量
git shortlog -sn --since="开始日期"

# 查看详细变更
git log --stat --since="开始日期"

# 查看某人的提交
git log --author="开发者名" --oneline
```

## ⚖️ 工作量合理性评估参考

### 常见功能模块参考工时

| 功能类型 | 简单 | 中等 | 复杂 |
|---------|------|------|------|
| 单表CRUD | 0.5-1天 | 1-2天 | 2-3天 |
| 列表查询+分页 | 0.5天 | 1天 | 1-2天 |
| 报表统计 | 1-2天 | 3-5天 | 5-10天 |
| 流程审批 | 3-5天 | 5-10天 | 10-20天 |
| 第三方接口对接 | 1-2天 | 3-5天 | 5-10天 |
| 用户权限系统 | 3-5天 | 5-10天 | 10-20天 |

> ⚠️ 以上仅供参考，实际工时取决于具体需求复杂度

### 隐性工作占比参考

| 工作类型 | 占比建议 |
|---------|---------|
| 编码开发 | 40-50% |
| 需求分析与沟通 | 10-15% |
| 设计与技术调研 | 10-15% |
| 测试与调试 | 15-20% |
| 文档编写 | 5-10% |
| 部署与交付 | 5-10% |

## 🚩 风险预警信号

以下情况需要特别关注：

- ⚠️ 代码提交频率过低或过于集中
- ⚠️ 功能进度与工时消耗不匹配
- ⚠️ 无法提供完整的工作日志
- ⚠️ 需求变更后未更新工时估算
- ⚠️ 拒绝定期演示进度
